# 电子宠物机 - 规则改进建议【2026年3月】

## 概述

根据当前项目的分析，提出以下几个方向的改进，以增强玩家的沉浸感和真实感，让宠物陪伴的体验更接近传统数码暴龙机。

---

## 一、孵化体验优化 ⭐⭐⭐

### 当前状态
- 等待倒计时完成
- 点击"孵化"随机破壳为幼年体

### 改进方案

#### 阶段1: 敲蛋壳（时间戳：孵化倒计时完成后）
```
✓ 倒计时完成 → 蛋壳出现细微裂纹
✓ 玩家需要点击"敲蛋"按钮，每次点击蛋壳增加一道裂纹
✓ 5-8次敲击后蛋壳完全破裂
✓ 最后按钮变为"破壳"，点击后幼年体露出
```

### 实现建议
- 在 `PetStage.egg` 增加中间态 `PetStage.hatching`
- 新增属性 `hatchCracks: int` 记录裂纹数
- 修改 UI: 在蛋的图像上叠加动态裂纹效果
- 触发日志: "蛋壳开始出现裂纹！", "蛋壳在摇晃！"

## 二、属性衰减与警告体系 ⭐⭐⭐⭐

### 当前状态
- 属性平稳衰减
- 危险值（<35）用红色显示

### 改进方案

#### 视觉警告升级
```
等级1 (>50) → 正常绿色
等级2 (35-50) → 橙色预警 + 轻微闪烁
等级2激活 (30-35) → 红色警告 + 界面轻微抖动
等级3 (<30) → 红色 + 急促闪烁 + 顶部警戒条
```

#### 属性衰减调整
```
当前配置 (示例):
- 饥饿: 每3分钟 -2
- 体力: 每2分钟 -1.5

建议调整为:
- 饥饿: 每2分钟 -3 (更快，强调喂食重要性)
- 心情: 每4分钟 -2
- 体力: 每2.5分钟 -2
(这会增加游戏紧凑感，提高交互频率)
```

#### 危险预警通知
```
属性即将危险 (进入35-40范围)
→ Toast提示: "🍽️ 宠物开始感到饥饿了..."
→ 同时在 eventText 中显示: "很饿"

进入生病前期 (多项属性<30, 尚未진入sick)
→ 触发新的 "虚弱" 状态
→ 宠物动画变为萎靡状态
→ Toast提示: "⚠️ 宠物需要照顾！"
```

### 实现建议
- 新增 `PetMood.warning` 状态
- 修改 UI: 属性栏根据值显示不同颜色和动画
- 在 `PetEngine.resolveTime()` 中增加状态检查逻辑
- 新增 `_getAttributeColor(double value)` 方法

## 三、战斗系统增强 ⭐⭐⭐⭐

### 当前状态
- 消耗体力和门票，结果随机

### 改进方案

#### 对战动画
```
1. 对方随机生成一只同级别宠物
2. 进入对战场景（可选：全屏动画或弹窗）
3. 双方像素宠物并排显示，轮流攻击
4. 根据双方战力计算伤害值
5. 3-5回合后决出胜负
```

#### 胜负概率优化
```
当前: 体力和门票满足即可参战

改进:
- 基础胜率 = (己方战力 / (己方战力 + 敌方战力)) * 100%
- 如果我方战力 ≥ 敌方战力 * 1.5: 95% 胜率
- 如果我方战力 ≤ 敌方战力 * 0.5: 5% 胜率
- 中间区间线性插值

==> 这样训练系统的价值就体现了，不能无脑对战
```

#### 战斗奖励
```
胜利:
- 战力 +1-3
- 纪律 +1
- 心情 +5
- wins +1
- 精英进化判定刷新

失败:
- 心情 -5
- 纪律 -1
- 可能触发"灰心"状态（心情大幅下降）
```

### 实现建议
- 创建 `BattleScreen` Widget
- 新增 `BattleEnemy` 类生成敌方宠物
- 修改 `applyAction(battle)` 计算逻辑
- 增加对战日志记录

## 四、隐藏分支进化触发 ⭐⭐⭐⭐⭐

### 当前状态
- 普通分支: 通过年龄、属性、胜场计算
- 秘码触发分支: 输入 `PROTO-MON`

### 改进方案：特殊条件进化

#### 冷漠线：宠物独立成长
```
触发条件:
1. 孵化后4小时内，总互动次数 < 5
2. 或：进入幼年期后，连续3小时无任何操作

进化后:
- 宠物形态改变（眯眼/冷漠表情）
- 名字后缀: "冷"
- 属性: 战力+15, 纪律+10, 心情-20
- 最终进化: 冷血战士形态

进化分支线: 冷漠 → 孤独 → 绝望 (隐藏最终形态)
(寓意：不照顾就会冷漠，最终离开)
```

#### 修行线：高强度训练
```
触发条件:
1. 连续8次训练操作（不间断）
2. 且纪律值 ≥ 70
3. 体力恢复循环 ≥ 3次

进化后:
- 宠物形态改变（闭眼冥想姿态）
- 名字后缀: "修" / "道"
- 属性: 战力+20, 纪律+15
- 对战胜率 +10%
- 最终形态: 修行大师/返璞归真

含义：通过课程化的训练，达到更高境界
```

#### 恋爱线：持续陪伴
```
触发条件:
1. 连续3天每天交互 ≥ 20次
2. 心情值始终 > 60
3. 卫生值始终 > 70

累积值: 陪伴度 (0-100)
- 每次正面互动 +1-2
- 每次负面事件 -1

进化后:
- 宠物形态改变（闪闪发光/羞涩表情）
- 展示特殊互动选项: "陪伴" / "交心"
- 最终形态: 伙伴/知己 (超稀有)

含义：通过持续用心照顾，建立深厚羁绊
```

### 实现建议
- 在 `PetSnapshot` 增加追踪变量:
  ```dart
  int interactionCount;        // 孵化后总交互数
  int trainingStreak;          // 连续训练次数
  int companionshipScore;      // 陪伴积分
  DateTime lastInteractionAt;  // 上次互动时间
  ```
- 在 `evolution()` 函数中增加触发检查
- 新增分支视觉标记（小图标/背景色）

## 五、后台悬浮窗体验 ⭐⭐⭐

### 当前状态
- 仅在主界面预览

### 改进方案

#### macOS 实现
```swift
// 游戏运行时：悬浮窗隐藏
// 游戏退出时：创建透明 always-on-top 小窗

- 窗口大小: 140x100 px (宠物像素充分显示)
- 位置：屏幕右下角
- 鼠标悬停：显示宠物名字 + 状态
- 点击：重新启动游戏（读取存档）
- 右键菜单：关闭悬浮窗

每秒更新一次宠物状态（通过 SharedPreferences 或文件共享）
```

#### 多窗口宠物
```
长期目标：支持多个宠物并行显示
- 用户可在应用内创建新宠物
- 后台独立显示每只宠物
- 点击任意悬浮窗启动对应宠物的游戏实例
```

### 实现建议
已在 Dart 层创建：
- `FloatingWindowManager` - 调用原生方法
- `AppLifecycleManager` - 监听应用生命周期

待完成（macOS 原生）：
- 在 `macos/Runner/MainFlutterWindow.swift` 中添加浮窗处理
- 创建 `FloatingWindowController` 管理独立窗口
- 使用 native Method Channel 通信

## 六、时间制系统细化 ⭐⭐⭐

### 当前状态
- 离线补算模式
- 属性定时衰减

### 改进方案

#### 真实时间里程碑
```
进阶功能：记录重要时间点

- 首次孵化时间
- 首次进化时间
- 总存活天数
- 最长陪伴连续日数
- 战斗总胜场时间分布

→ 在属性面板/图鉴显示为 "养成日志" 而非生冷数据
```

#### 季节概念（可选）
```
基于当前日期:
- 春季 (3-5月): 属性衰减-20%, 探索获得加成
- 夏季 (6-8月): 战斗加成, 湿度影响 (衣生)
- 秋季 (9-11月): 食物更香, 心情恢复快
- 冬季 (12-2月): 尖刻天气, 需要更多休息

→ 增加玩法变化性和重复性尝试的动力
```

## 总体设计目标

| 维度 | 当前 | 目标 |
|-----|-----|-----|
| **交互频率** | 中等 | 高（增加紧迫感） |
| **决策深度** | 低（无脑操作） | 中（策略化） |
| **沉浸感** | 中 | 高（视觉反馈、动画） |
| **重复性** | 中 | 高（多分支、隐藏） |
| **真实感** | 中 | 高（时间制、后台） |
| **养成质感** | 中 | 高（进化意义大） |

## 备注

这些建议的核心理念：
1. **让玩家感受到 "后果"** - 不照顾 → 变冷漠，认真训练 → 变强
2. **增加 "惊喜" 发现** - 隐藏分支、秘码触发、特殊条件
3. **强化 "真实感"** - 离线补算、后台窗口、时间标记
4. **提高 "游戏质感"** - 动画、音效（后续）、视觉反馈

期待看到电子宠物机逐步演进为一个真正有灵魂的陪伴应用。🎮✨
